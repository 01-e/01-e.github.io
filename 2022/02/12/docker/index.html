简单使用docker

docker 是什么

docker是一种容器技术，他解决软件跨环境迁移的问题

docker 服务命令

docker启动：systemctl start docker

docker停止：systemctl stop docker

docker状态：systemctl status docker

docker重启：systemctl restart docker

docker开机启动：systemctl enable docker

docker镜像命令

查看docker镜像：docker images

查看所有docker镜像ID：docker images -q

搜索网络中的docker镜像：docker search 镜像名称

从远程下载镜像：docker pull 镜像名称:版本号

可以在https://hub.docker.com/上查找docker镜像版本号

删除docker镜像：docker rmi 容器ID、docker rmi 镜像名称:版本号

删除本地所有docker镜像：docker rmi `docker images -q`

docker容器命令

查看正在运行的docker容器：docker ps

查看所有docker容器：docker ps -a

创建并启动容器：docker run 参数

  • -i：保持容器运行
  • -t：为容器分配伪终端
  • --name=容器名：为创建的容器命名
    • 例如：docker run -it --name=c1 redis:6.2.6
    • 用-it创建容器，创建完会自动进入容器，exit退出后容器不再运行
  • -d：以守护（后台）形式运行容器
    • 例如：docker run -id --name=c2 redis:6.2.6
    • 用id创建容器，创建完不进入容器，而是在后台运行容器，退出后不影响容器的运行
    • docker exec -it c2 /bin/bash：进入用-id创建的容器

进入容器：docker exec 参数

启动容器：docker start 容器名

停止容器：docker stop 容器名

删除容器：docker rm 容器名/容器ID

  • 删除容器前必须停止容器

查看所有容器ID：docker ps -aq

查看容器信息：docker inspect 容器名/容器ID

docker容器的数据卷

  • 数据卷是宿主机（服务器）中的一个目录或者文件

  • 当容器目录和数据卷目录绑定后，对方的修改会立即同步

  • 一个数据卷可以被多个容器同时挂载

  • 一个容器也可以被挂载多个数据卷

配置数据卷：docker run ... -v 宿主机目录（文件）:容器内目录（文件）...

  • docker run -it --name=c1 -v /root/data:/root/data_container mysql:8.0 /bin/bash

数据卷容器

多容器进行数据交换：

  1. 多个容器挂载同一个数据卷
  2. 数据卷容器

创建启动c3数据卷容器，使用-v参数设置数据卷：docker run -it --name=c3 -v /volume centos:8 /bin/bash

创建启动c1、c2容器，使用–volumes-from参数设置数据卷：docker run -it --name=c1 --volume-from c3 centos:8 /bin/bash、docker run -it --name=c2 --volume-from c3 centos:8 /bin/bash
